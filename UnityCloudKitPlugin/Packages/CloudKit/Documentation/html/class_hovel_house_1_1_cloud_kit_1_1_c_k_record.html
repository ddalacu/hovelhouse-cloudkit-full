Detailed Description

A single record inside a CloudKit database

Everything stored in a database is stored as a CKRecord. All CKRecords are of a 'type' which you specify during construction. Records contain one or more fields of a certain limited set of types: strings, bools, ints, doubles, dates, locations, references or assets. All records are uniquely identified by a recordID which you may (optionally) specify. If you don't specify a record id, one is automatically generated for you. Once a record is saved to the database it will contain useful metadata, such as creation time, last modified time, and change tokens.
